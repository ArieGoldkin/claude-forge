The 8-Phase Experiment Loop

Each iteration modifies code, measures a metric, and keeps or discards the change. Git commits before every evaluation ensure clean rollback on regression.

Phase Details

1. Setup -- Parse args or .experiment/config.yaml, validate target exists in git repo, dry-run metric command, create results directory.

2. Baseline -- Run metric command on unmodified code, record iteration 0 as baseline.

3. Hypothesize -- Read target code + prior results, formulate a specific hypothesis before modifying anything.

4. Modify -- Apply one atomic change within scope, commit with experiment message.

5. Evaluate -- Run metric command, parse numeric result, compare to current best.

6. Decide -- Keep improvement or git-reset regression. Simplicity tiebreaker: <0.1% gain with >20 lines changed = discard.

7. Log -- Append row to .experiment/results/{name}.tsv with all fields.

8. Iterate? -- Check stop conditions (goal, budget, time, stuck), loop or generate report.

Common Use Cases

API Latency

Reduce mean response time below a target threshold.

metric: mean latency minimize

/experiment src/api/handler.ts \
  --metric "npm run bench -- --json \
    | jq '.results[0].mean'" \
  --minimize --goal 100 --unit ms

Test Coverage

Push line coverage toward a target percentage.

metric: line coverage % maximize

/experiment src/ \
  --metric "npm test -- --coverage \
    | grep Lines | awk '{print $2}'" \
  --maximize --goal 90 --unit %

Bundle Size

Minimize production bundle output size.

metric: bundle bytes minimize

/experiment src/ \
  --metric "npm run build && \
    stat -f%z dist/bundle.js" \
  --minimize --iterations 15 \
  --unit bytes

ML Training Loss

Tune hyperparameters to reduce training loss.

metric: final loss minimize

/experiment train/config.py \
  --metric "python train.py \
    | tail -1 | awk '{print $NF}'" \
  --minimize --goal 0.15 \
  --unit loss --iterations 20

Lighthouse Score

Improve web performance to reach a target score.

metric: perf score maximize

/experiment src/pages/ \
  --metric "npx lighthouse \
    http://localhost:3000 \
    --output json | jq \
    '.categories.performance.score
     * 100'" \
  --maximize --goal 95

Configuration Reference

Configure via inline flags or .experiment/config.yaml. Inline args take precedence over file values.

Example .experiment/config.yaml

# Experiment identity
name: reduce-api-latency
description: Optimize the API handler to reduce mean response time
target: src/api/handler.ts

# Files the agent must not modify
readonly:
  - src/api/types.ts
  - tests/

# What to measure and which direction is better
metric:
  command: "npm run bench -- --json"
  extract: "jq '.results[0].mean'"
  direction: minimize
  unit: ms
  goal: 100

# Iteration and time limits
budget:
  max_iterations: 15
  max_minutes: 60
  iteration_timeout_minutes: 5
  checkpoint_every: 5

# Guide the agent's approach
hints:
  - "Consider connection pooling optimizations"
  - "The database query in lines 45-60 may be N+1"
  - "Do not change the public API signature"

Safety Guardrails

Git Rollback Visualization

Every modification is committed before the metric runs. Regressions are rolled back to the previous commit.

Commit -- experiment: iteration N - description
Evaluate -- run metric command, parse result
Improved? Keep commit, update best value
-- or --
Regressed? git reset --hard HEAD~1 -- change never persisted
Next iteration -- always starts from last known-good state

Guardrails Checklist

+
Readonly enforcement -- git diff --name-only checked against readonly list; violations revert immediately
+
Correctness gate -- metric command crash or non-numeric output triggers automatic revert
+
Iteration budget -- max_iterations (default 20, cap 100) prevents runaway loops
+
Time budget -- max_minutes (default 60, cap 480) wall-clock limit
+
Stuck detection -- 5 consecutive discards triggers strategy reset; 6 consecutive stops
+
Human checkpoint -- optional pause every N iterations for review (checkpoint_every)
+
Scope containment -- only files within target path may be modified
+
Commit-before-evaluate -- every change committed before metric runs, ensuring clean revert
+
Clean git state -- stash uncommitted changes before starting; restore on completion
+
Simplicity tiebreaker -- <0.1% gain with >20 lines changed = discard
